Building Layouts with Handlebars

February 3, 2026
layoutshandlebars

Mikser layouts are Handlebars templates with a double extension. The first extension defines the output format (.html, .xml, …), the second names the template engine (.hbs). This post covers the most useful patterns.

Accessing document data

Inside every layout the current document is available as document:

<!-- front-matter fields -->
{{document.meta.title}}
{{document.meta.date}}

<!-- rendered body content -->
{{{markdown document.content}}}

Use triple braces {{{…}}} when you want to output raw HTML — otherwise Handlebars will escape it.

Conditional blocks

{{#if document.meta.author}}
  <span>By {{document.meta.author}}</span>
{{/if}}

Iterating over arrays

Front-matter arrays (like tags) can be looped with #each:

{{#each document.meta.tags}}
  <span class="tag">{{this}}</span>
{{/each}}

The href helper

The render-href plugin registers an href() Handlebars helper that resolves any entity from the sitemap by its href value. This is the right way to create links between pages — the helper returns the entity's actual output destination, so links stay correct even when clean URLs are enabled:

{{#with (href '/blog/getting-started')}}
  <a href="{{link}}">{{meta.title}}</a>
{{/with}}

To resolve a specific language variant, pass the lang as a second argument: {{href '/blog/getting-started' 'bg'}}.

Resource and asset helpers

The render-resource and render-asset plugins register Handlebars helpers that resolve remote media through the resource and asset plugins:

<!-- original image URL resolved through the resource library -->
<img src="../public/{{get 'link' (resource document.meta.image)}}">

<!-- processed WebP via an asset preset -->
<img src="../public/{{asset 'small-image' (get 'name' (resource document.meta.image)) 'webp'}}">

PDF layouts

Add -pdf to the format segment of the layout filename to enable PDF post-processing:

layouts/
└── report.html-pdf.hbs   # renders HTML, then Puppeteer converts to PDF

The post-pdf plugin launches a headless Chromium instance, navigates to the rendered HTML file, and prints it to PDF using the browser's built-in print engine — so you get full CSS, page breaks, and print media queries for free.

The Puppeteer browser instance is shared across all PDF jobs in a single run and kept alive for 60 seconds after the build completes, so rapid watch-mode rebuilds reuse the same instance without relaunching.
